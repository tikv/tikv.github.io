Creates the cluster with cluster ID, node, stores and first Region. If the cluster is already bootstrapped, return ClusterBootstrapped error. When a node starts, if it finds nothing in the node and cluster is not bootstrapped, it begins to create node, stores, first Region and then call bootstrap_cluster to let PD know it. It may happen that multi nodes start at same time to try to bootstrap, but only one can succeed, while others will fail and must remove their created local Region data themselves. Read more
